# Plan 11-03: Shimmer Skeleton Integration & Text Scaling Checks

## Objective

Replace all `CircularProgressIndicator` loading states with premium Shimmer-based skeleton cards that match the exact layout shapes of the content they replace. Additionally, audit all text widgets to ensure they are wrapped in `Flexible` or `Expanded` where needed, preventing `RenderFlex overflowed` errors when `textScaleFactor` is increased on devices.

## must_haves

- [ ] Reusable `SkeletonCard` and `SkeletonList` widgets exist in `core/widgets/`.
- [ ] Teacher dashboard replaces CircularProgressIndicator with SkeletonList during loading.
- [ ] Parent dashboard replaces CircularProgressIndicator with SkeletonCard during loading.
- [ ] Academic analytics view uses skeleton during chart data loading.
- [ ] No `RenderFlex overflowed` warnings when device text scale is set to 1.5x.

<read_first>

<action>
1. Create `mobile_app/lib/core/widgets/skeleton_card.dart`:
   ```dart
   import 'package:flutter/material.dart';
   import 'package:shimmer/shimmer.dart';

   class SkeletonCard extends StatelessWidget {
     final double height;
     final double width;
     final double borderRadius;

     const SkeletonCard({
       super.key,
       this.height = 120,
       this.width = double.infinity,
       this.borderRadius = 24,
     });

     @override
     Widget build(BuildContext context) {
       final baseColor = Theme.of(context).colorScheme.surfaceVariant;
       final highlightColor = Theme.of(context).colorScheme.surface;

       return Shimmer.fromColors(
         baseColor: baseColor,
         highlightColor: highlightColor,
         child: Container(
           height: height,
           width: width,
           margin: const EdgeInsets.symmetric(horizontal: 20, vertical: 8),
           decoration: BoxDecoration(
             color: baseColor,
             borderRadius: BorderRadius.circular(borderRadius),
           ),
         ),
       );
     }
   }
   ```

2. Create `mobile_app/lib/core/widgets/skeleton_list.dart`:
   ```dart
   import 'package:flutter/material.dart';
   import 'skeleton_card.dart';

   class SkeletonList extends StatelessWidget {
     final int itemCount;
     final double itemHeight;

     const SkeletonList({
       super.key,
       this.itemCount = 4,
       this.itemHeight = 80,
     });

     @override
     Widget build(BuildContext context) {
       return Column(
         children: List.generate(
           itemCount,
           (_) => SkeletonCard(height: itemHeight),
         ),
       );
     }
   }
   ```
</action>

- SkeletonCard reads `Theme.of(context).colorScheme.surfaceVariant` for base color
- SkeletonList renders a `Column` of `SkeletonCard` widgets
</acceptance_criteria>

<task id="11-03-04">
<title>Text scaling audit and Flexible wrapping</title>
<read_first>
- mobile_app/lib/features/teacher/views/teacher_dashboard_view.dart
- mobile_app/lib/features/parent/views/parent_dashboard_view.dart
- mobile_app/lib/features/parent/widgets/hero_attendance_card.dart
</read_first>
<action>
Audit all `Row` widgets containing `Text` children. Any `Text` widget inside a `Row` that is NOT wrapped in `Flexible` or `Expanded` and could overflow at 1.5x text scale must be wrapped:

Pattern to fix:
```dart
// BEFORE (vulnerable to overflow)
Row(children: [Icon(...), Text('Long text here')])

// AFTER (safe)
Row(children: [Icon(...), Flexible(child: Text('Long text here', overflow: TextOverflow.ellipsis))])
```

Apply this fix in:
1. `teacher_dashboard_view.dart` — header rows, timeline rows
2. `parent_dashboard_view.dart` — header, notice bar text
3. `hero_attendance_card.dart` — stat row labels

Add `maxLines: 1` and `overflow: TextOverflow.ellipsis` to all wrapped Text widgets for graceful truncation.
</action>
<acceptance_criteria>
- teacher_dashboard_view.dart contains `Flexible(child: Text(` pattern
- parent_dashboard_view.dart contains `Flexible(child: Text(` or `Expanded(child: Text(` pattern
- hero_attendance_card.dart contains `overflow: TextOverflow.ellipsis`
- No bare `Text` children exist directly inside `Row` without Flexible/Expanded wrapping (in the listed files)
</acceptance_criteria>
</task>
